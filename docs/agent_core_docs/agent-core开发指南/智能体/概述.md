openJiuwen致力于提供高效、灵活的Agent应用开发支持，帮助用户快速构建智能化、自动化的Agent应用系统，轻松应对各类复杂任务。目前，openJiuwen提供了ReActAgent和WorkflowAgent这两种预置智能体，提供了丰富的功能和灵活的开发选项，满足用户不同场景下的智能需求。

# ReActAgent

ReActAgent是一种遵循ReAct模式的Agent，通过 “思考（Thought）→ 行动（Action）→ 观察（Observe）”的循环迭代完成用户任务。首先，模型进行逻辑推理并规划，选择并调用合适的工具（如检索、数据库、第三方 API、代码执行等）执行具体的操作；其次，根据执行结果观察与反馈，不断调整策略，优化推理路径，直至达成任务目标或获得最终答案。其强大的多轮推理与自我修正能力，使ReActAgent具备动态决策能力且能够灵活应对环境变化，适用于需要复杂推理和策略调整的多样化任务场景。

## 支持的场景

- **非流式调用**：调用`ReActAgent`的`invoke`接口，输入用户请求；输出智能体调用若干次工具后，对用户请求的整体答复。
- **流式调用**：调用`ReActAgent`的`stream`接口，输入用户请求；流式输出`ReActAgent`对用户请求的整体答复的数据帧。

## 应用场景

ReActAgent适用于需要实时响应、动态决策和交互能力的场景，具有高度的灵活性和自适应能力，适合处理非结构化或变化频繁的任务。

- 智能客服：实时理解用户意图，提供个性化回复，不仅解答常见问题，还能根据用户情绪、上下文进行推理与引导，提升服务体验。
- 智能家居：根据传感器数据实时调整家居环境，如自动调节室温、灯光亮度、播放背景音乐等，提升居住舒适度与节能效率。

# WorkflowAgent

WorkflowAgent是一种专注于多步骤、任务导向的流程自动化Agent，通过严格遵循用户预定义的任务流程高效地执行复杂任务。用户可预先设定清晰的任务步骤、执行条件及角色分工，将任务拆解为多个可执行的子任务或工具，并通过组件间的拓扑连接与数据传递，逐步推进整个工作流，最终输出预期结果。其侧重于基于预设流程实现任务的规范化与高效化执行，适用于任务结构清晰、可分解为多个步骤的场景。

## 支持的场景

- **非流式调用**：调用`WorkflowAgent`的`invoke`接口，输入工作流依赖的输入参数；输出工作流的执行结果。
- **流式调用**：调用`WorkflowAgent`的`stream`接口，同样输入工作流依赖的输入参数；流式输出若干个数据帧，包含`WorkflowAgent`关联工作流的流式输出数据帧、调测信息数据帧，以及`WorkflowAgent`最终执行结果的数据帧。
- **人机交互**：当`WorkflowAgent`关联的工作流中包含带中断恢复能力的组件（例如，预置组件中的提问器组件），则可能触发人机交互流程。根据`WorkflowAgent`的返回结果中是否包含向用户追问的问题，能够判断是否触发了人机交互的流程。一旦触发人机交互流程，开发者则需要将用户的反馈信息作为`WorkflowAgent`的输入，再次调用`WorkflowAgent`的执行流程，直至`WorkflowAgent`返回了最终的执行结果。

## 应用场景

WorkflowAgent专注于流程自动化与任务协同，适用于结构清晰、步骤明确、重复性高的任务场景，强调任务的高效执行与系统间的无缝衔接。

- 项目管理：进行任务分配、进度跟踪、资源调度等，确保项目按照既定目标和时间表顺利进行。
- 企业自动化：实现审批流程、数据流转、报表生成等日常操作的自动化，减少人工干预，降低出错率，提高运营效率。
- 数据处理与分析：整合多源数据，自动清洗、分析与可视化，生成结构化报告、分析结果，并将其转化为决策支持。
